# Frontend

## Опис

Frontend для проєкту учнівського самоврядування **ЕРА** - міністерства культури та мистецтва Березанського ліцею №3. Ідея проста: зробити зручний хаб для учнів та вчителів, де можна буде

- отримати розклад уроків*
- дізнатись найближчі шкільні події

\* - знаходиться в розробці

> Проєкт тільки почав рости, але буде потужно :P

## Функціонал 

- **Авторизації в адмін-панель**
    Працює через backend, логін/пароль зберігаються в PostgreSQL.

- **Роутинг**
    Нормальна навігація між сторінками.

- **HTTP запити до бекенду**

## Установка та запуск

> Інструкція передбачає встановлений npm.

```bash
npm install
npm start
```

## Архітектура

```
lyceum-frontend/
├── public/                 # Статичні файли: index.html, іконки, маніфести — основа рендера
├── src/                    # Весь вихідний код застосунку
│   ├── api/                # Запити на бекенд, axios-клієнт, ендпоінти
│   ├── components/         # Повторно використовувані UI-компоненти
│   ├── data/               # Локальні дані, JSON-файли, мокові списки
│   ├── hooks/              # Кастомні React-хуки
│   ├── pages/              # Сторінки (routes): Home, AdminDashboard…
│   ├── router/             # Налаштування маршрутизації
│   ├── services/           # Бізнес-логіка, робота з локальним storage та іншими сервісами
│   ├── types/              # TypeScript типи та інтерфейси
│   ├── App.tsx             # Головний компонент застосунку
│   ├── index.tsx           # Точка входу: рендер React у DOM
│   └── index.css           # Глобальні стилі
├── .gitignore              # Файли/папки, які не потрапляють у git
├── LICENSE                 # Ліцензія проєкту
├── package.json            # Список залежностей + скрипти npm
├── package-lock.json       # Лочить версії всіх залежностей
├── README.md               # Основна документація для проєкту
└── tsconfig.json           # Налаштування TypeScript
```

Фронт підіймається до localhost, бек треба запустити окремо.

## Backend

Фронтенд працює разом із backend-сервесом (Spring + PostgreSQL)

На беку вже реалізовано:

- авторизація через JWT
- CRUD для подій ліцею
- зберігання адмінів у PostgreSQL
- декілька ендпоінтів
- налаштовані конфіги безпеки

Поки що там базові речі, бо проект на стадії зародження, але розганяється.

## Деплой
